## Writing Strategy

### Recommended Order
1. **Method** (easiest, based on preregistration)
2. **Results** (structured around your analysis)
3. **Introduction** (sets up the study)
4. **Literature Review** (synthesizes existing work)
5. **Discussion** (interprets findings)
6. **Conclusion** (summarizes)
7. **Abstract** (write last)
8. **References** (compile throughout)

### Key Points to Emphasize
1. **Preregistration** - this is a strength
2. **Counterintuitive finding** - personalization deters delegation
3. **Economic paradox** - participants don't choose what's optimal
4. **Benefits of delegation** - 2/3 of self-choices are dominated
5. **Transparency** - be clear about mixed support for H1
6. **Theoretical contribution** - when personalization helps vs. hurts

### Style Guidelines (CHB)
- Use clear, accessible language
- Avoid jargon where possible
- Report effect sizes, not just p-values
- Include confidence intervals
- Follow APA citation style
- Use active voice where appropriate
- Keep paragraphs focused (3-5 sentences)
- Use transitions between sections

## Section-Specific Tips

### Abstract
- [ ] Background (2-3 sentences)
- [ ] Objective (1 sentence)
- [ ] Method (2-3 sentences)
- [ ] Results (3-4 sentences with key statistics)
- [ ] Conclusion (2-3 sentences)
- [ ] Keywords (5-7 terms)
- [ ] Word count: ~200 words

### Introduction
- [ ] Start broad (AI in decision-making)
- [ ] Narrow to problem (delegation to algorithms)
- [ ] Identify gap (effect of algorithm type)
- [ ] State contribution (preregistered experiment)
- [ ] Present hypotheses (H1 and H2)
- [ ] Provide roadmap
- [ ] Word count: ~800 words

### Literature Review
- [ ] Start with algorithm aversion vs. appreciation
- [ ] Move to trust and reliance
- [ ] Discuss personalization
- [ ] Cover delegation in finance
- [ ] Briefly mention individual differences
- [ ] Position your study
- [ ] Word count: ~1200 words

- [ ] Participants (recruitment, sample, exclusions)
- [ ] Experimental design (treatments, structure)
- [ ] Procedure (step-by-step)
- [ ] Measures (primary and secondary)
- [ ] Statistical analysis (preregistered)

### Results
- [ ] Descriptive statistics

- [ ] Benefits of delegation (2/3 dominated choices)

- [ ] Win-stay/lose-shift patterns
- [ ] Additional analyses

### Discussion
- [ ] Main findings (personalization deters delegation)

### Conclusion

## Common Pitfalls to Avoid

### Content Issues
- [ ] Don't overstate results - be precise about what was found
- [ ] Don't hide mixed support for H1 - be transparent
- [ ] Don't ignore economic perspective - explain why personalized algorithm should be preferred
- [ ] Don't forget to emphasize benefits of delegation (2/3 dominated choices)
- [ ] Don't make unsupported claims - stick to what data shows

### Writing Issues
- [ ] Don't use passive voice excessively
- [ ] Don't use jargon without explanation
- [ ] Don't write overly long paragraphs
- [ ] Don't forget transitions between sections
- [ ] Don't repeat the same information

### Statistical Issues
- [ ] Don't report only p-values - include effect sizes
- [ ] Don't confuse statistical significance with practical significance
- [ ] Don't ignore confidence intervals
- [ ] Don't forget to report sample sizes
- [ ] Don't make claims not supported by data
